# HookEuicc

特定のアプリでの eSIM 要件をバイパスし、eSIM アクティベーションコードを取得を可能にします。

eSIM のアクティベーションコードを直接表示しないアプリの場合は、モジュールがコードを自動的にクリップボードにコピーします。

## 更なる機能

### 非 eUICC モード

eUICC [ISD-R] が排他的アクセス権を持つ特定のデバイスの問題を修正します。 

**一例:** 一部の Qualcomm 製デバイスで自作のハンダ付けによる ST33 チップを使用している場合。

> [!NOTE]
> スコープで `com.android.phone` を選択して再起動する必要があります。 
> ホットスワップは非対応です。電源投入または再起動前に eUICC が挿入されていることを確認してください。

### OMAPI バイパス

ARA (アクセスルールアプリケーション) と ARF (アクセスルールファイル) の制限をバイパスします。

通常は、ARA が適用されていないカードに OMAPI アクセス権限を付与するために使用されます。

> [!NOTE]
> スコープで `com.android.se` を選択して再起動 (またはこちらを実行: `su -c killall com.android.se`) してください。
> 問題が発生していない限りは、この機能を有効化しないでください。
